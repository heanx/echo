# Echo Phase 4 产品闭环与细节打磨

更新时间：2026-05-25

Phase 4 的目标是把 Echo 从“功能能演示”推进到“核心路径稳定、交互连续、数据关系清楚、页面可维护”的课程项目收口状态。本阶段重点不是继续堆新概念，而是补齐歌单、评论、上传发布流、搜索和主内容区局部导航这些会直接影响使用体验的闭环。

## 1. 当前状态

截至 2026-05-25，Phase 4 主干已经完成：

- 歌单功能按设计规范 V2 完成核心建模和页面体验。
- 评论体验补齐分页、排序、折叠、删除 / 撤回和个人主页联动。
- 上传音乐页从数据库字段式表单重构为产品级发布流程。
- 封面、音频、歌词上传都支持前置校验和拖拽 / 点击选择。
- 主内容区核心导航改为局部替换，尽量避免底部播放器被刷新销毁。
- 搜索系统支持歌单结果，搜索建议和搜索结果可进入歌单。

## 2. 歌单闭环

### 2.1 数据模型

歌单模型采用“可见性、协作、系统类型分离”的设计：

- `visibility`：只表示公开或私密。
- `collaboration_mode`：预留协作能力，不混入可见性。
- `playlist_type`：区分普通歌单和系统默认“我喜欢的音乐”。
- `PlaylistTrack`：承载歌曲顺序、添加者和去重约束。
- `PlaylistFavorite`：保存用户收藏歌单关系。

默认“我喜欢的音乐”作为用户的默认收藏歌单，不再是左侧音乐库里的孤立入口，而是并入“我的歌单”并固定置顶。

### 2.2 路由约定

旧的顶层歌单路由已经移除，歌单统一挂到用户空间：

```text
/users/<username>/playlist/
/users/<username>/playlist/create/
/users/<username>/playlist/<id>/
/users/<username>/playlist/<id>/edit/
```

这样可以减少项目路由复杂度，也更贴近“某个用户的歌单空间”的产品语义。

### 2.3 页面与交互

已完成：

- 左侧音乐库新增“我的歌单”和“收藏的歌单”。
- 歌单中心展示默认喜欢歌单、用户普通歌单和收藏歌单。
- 歌单详情支持封面、歌曲列表、播放全部、更多菜单、批量管理和歌曲排序。
- 歌单封面支持上传、修改和替换清理。
- 歌单搜索结果接入 `/search/`。
- 歌单相关入口尽量使用主内容区局部替换，避免中断底部播放器。

后续仍可继续打磨：

- 多选操作的视觉反馈。
- 更完整的拖拽排序动画。
- 协作歌单邀请和成员管理。

## 3. 评论闭环

评论系统已经从“可提交”推进到更接近真实社区体验：

- 评论列表分页。
- 保留热门、最新、问题筛选。
- 回复默认折叠展示。
- 评论本人、作品作者和管理员可删除 / 撤回。
- 个人主页最近评论可跳转到对应讨论位置。
- 评论回复和点赞继续触发通知。

当前仍建议继续关注：

- 删除评论后的前端计数同步。
- 更完整的回复链展开体验。
- 通知偏好和批量已读。

## 4. 上传发布流

上传音乐页已经重构为产品级流程：

```text
上传音频 -> 自动解析信息 -> 填写作品信息 -> 设置封面 -> 设置歌词 -> 保存音乐
```

### 4.1 音频上传

已完成：

- 大号拖拽上传卡片。
- 空状态、拖拽悬停、读取中、解析中、解析成功、错误状态。
- 格式和 50MB 大小前置校验。
- 上传后展示文件名、格式、大小、时长。
- 移除“时长毫秒”输入框，改为只读歌曲时长。

### 4.2 封面设置

封面模块已经重构为稳定编辑卡片：

- 桌面端左侧固定 1:1 预览，右侧为操作区。
- 移动端单列顺序为预览、上传、状态操作、默认风格、裁剪控制。
- 上传区域、状态操作、默认封面风格、裁剪控制分组清晰。
- 裁剪控制保留结构位置，通过禁用和弱化避免高度跳动。
- 封面上传避免 `label for=file` 与自定义点击逻辑叠加导致白屏。

### 4.3 歌词设置

已完成：

- 音频上传页和独立歌词上传页都支持歌词文件选择。
- 歌词文件入口避免直接 `label for=file` 触发，改为显式 picker。
- 支持 `.lrc` / `.txt` 文件读取、分列编辑和预听校对。
- 歌词语言从手输改为下拉选择，避免暴露 `zh-Hans` 等内部值。

2026-05-31 已补：

- 音频上传页和独立歌词上传页的歌词编辑器 JS 已抽成 `static/js/lyrics-editor.js`。
- 两个页面统一复用解析、序列化、逐行编辑、删除行和填入当前时间逻辑。

后续建议：

- 增加更细的导入错误提示和歌词时间轴校验。

## 5. 主内容区局部导航

为了保持底部播放器不中断，以下入口已经尽量改为只刷新 `#main-content`：

- 顶部搜索提交。
- 搜索建议和搜索结果链接。
- 左侧音乐库的全部、最新、我的歌单、收藏歌单和歌单详情。
- 创建菜单中的上传音乐、歌单、共享合辑、上传歌词。
- 歌单详情里的歌曲详情、评论入口和收藏者主页。
- 歌曲右键菜单中的查看评论和查看歌曲。

注意：后续新增主内容页面时，应优先使用 `data-main-nav` 或 HTMX `hx-select="#main-content"`，避免整页刷新。
