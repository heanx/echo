# Echo 阶段说明书

更新时间：2026-05-13

## 1. 阶段定位

当前阶段目标是把 Echo 从“课程作业级功能原型”推进到“工程基础较稳、交互闭环更完整”的版本。

项目仍定位为轻量级音频分享社区，技术栈保持：
- Django：路由、ORM、上传、业务逻辑
- SQLite：本地开发与演示
- Tailwind CSS：界面样式
- HTMX + 原生 JavaScript：中部内容局部刷新与全局播放器状态管理

## 2. 当前已完成能力

### 2.1 应用壳与主流程

- 顶部导航、左侧音乐库、中部主内容、右侧播放上下文、底部播放器已形成统一应用壳
- 首页、歌词页、评论页可在 `#main-content` 局部切换
- 底部播放器常驻，切换中部内容不会打断播放状态

### 2.2 音频与上传

- 支持常见音频格式上传：`.mp3`、`.wav`、`.ogg`、`.m4a`、`.aac`、`.flac`、`.webm`
- 支持封面图片上传与主题色 fallback
- 支持轻量 ID3 信息读取，用于自动补全标题、作者、专辑
- 播放量上报接口已落地：`POST /tracks/<track_id>/play/`

### 2.3 歌词系统

- 支持歌词上传页和音频上传页中的歌词录入
- 支持 LRC / TXT 文本解析
- 支持 `available / instrumental / pending` 三种状态
- 歌词页支持高亮、点击 seek、滚动跟随
- 歌词背景已改为基于封面主色 / 主题色的纯色方案

### 2.4 评论系统

- 支持按曲目读取评论
- 评论响应禁用缓存，避免切歌后复用旧 HTML
- 评论提交支持 HTMX partial 刷新
- 后端已预留评论回复、点赞、问题评论、置顶等字段

### 2.5 播放器状态与队列

- 已支持顺序播放、列表循环、单曲循环、随机播放
- 已支持上一首、下一首、播放结束自动下一首
- 已支持播放模式本地持久化
- 已支持播放进度、当前曲目、音量本地持久化
- 单曲循环已同步到原生 `audio.loop`，避免播放队列重构后影响单曲循环行为
- 2026-05-13 修复：`selectTrack()` 切换音频源后重新同步 `audio.loop`，防止浏览器媒体加载算法重置 loop 属性；边界场景（列表首尾、随机池耗尽）单曲循环模式下改为回绕或重播当前曲目
- 已引入真实播放队列状态：
  - `playQueue`
  - `playQueueIndex`
  - `playQueueName`
- 首页推荐、首页最新、左侧最近播放、详情页、右侧播放列表等入口，已能为播放器建立明确队列来源

### 2.6 用户认证与资料

- 已接入 Django 注册、登录、退出
- 已新增 `UserProfile`：显示名、头像、简介
- 顶部导航已区分登录态与匿名态
- 上传音乐、上传歌词要求登录
- 上传作品会绑定到当前用户
- 登录页支持“保持登录”，密码保存交给浏览器密码管理器
- 注册页和资料设置页支持头像上传、文件名旁取消选择、默认放大预览、拖动裁剪位置和缩放
- 头像限制为 JPG / PNG / WEBP，最大 4MB；后端继续做文件头校验和 Pillow 压缩，数据库只保存文件路径

### 2.7 工程地基

- 中文乱码问题已清理
- 已有基础后端回归测试
- 已接入列表分页、空状态、全局错误页、请求失败 toast
- Django 配置已拆分为：
  - `echo_project.settings.base`
  - `echo_project.settings.dev`
  - `echo_project.settings.prod`
- 上传安全校验已增强：
  - 扩展名校验
  - 文件大小校验
  - 文件头魔数校验
  - 伪装文件与空文件拦截
