# Echo 阶段说明书

更新时间：2026-05-11

## 1. 阶段定位

本阶段目标是把 Echo 从早期功能原型推进到“大作业初步可用版本”。当前版本已经具备音频社区的核心演示闭环：用户可以浏览音乐、上传音频、播放歌曲、查看歌词、进入评论区、提交评论，并在播放时记录播放量。

项目仍然定位为“轻量级极客音频流媒体社区”，技术路线保持为：

- Django：后端路由、ORM、上传、业务查询。
- SQLite：本地课程演示与轻量持久化。
- Tailwind CSS：页面壳、响应式界面和视觉样式。
- HTMX + 原生 JavaScript：中部内容局部刷新和全局播放器状态管理。

## 2. 已完成成果

### 2.1 应用壳与交互体验

- 完成顶部导航、左侧音乐库、中部主内容、右侧播放上下文、底部播放器的基础应用壳。
- 首页、歌词页、评论页可以在 `#main-content` 中局部切换。
- 底部播放器常驻，切换中部内容时播放状态不随页面刷新丢失。
- 深浅色主题、侧栏折叠、右侧播放上下文、播放列表等基础交互已可用。

### 2.2 音频与媒体能力

- 支持上传常见音频格式：`.mp3`, `.wav`, `.ogg`, `.m4a`, `.aac`, `.flac`, `.webm`。
- 支持封面图片上传和封面 fallback 主题。
- 支持前后端轻量读取音频元数据，用于自动填充标题、作者等信息。
- 支持播放量上报接口：

```text
POST /tracks/<track_id>/play/
```

播放量更新使用 `F("plays") + 1`，避免并发覆盖。

### 2.3 歌词系统

- 支持歌词上传页和音乐上传页内的歌词录入。
- 支持 LRC / TXT 文本解析。
- 支持歌词状态：

```text
available
instrumental
pending
```

- 歌词展示页支持根据当前播放时间高亮歌词行。
- 歌词区已优化长句换行、字体、当前行样式和滚动行为。

### 2.4 评论系统

- 评论模型和查询逻辑已落地。
- 评论页支持按歌曲读取评论。
- 评论响应头禁用缓存，避免切歌后复用旧 HTML。
- 评论提交支持 HTMX partial 插入。
- 后端已经预留评论回复、点赞、问题评论、置顶评论等字段。

### 2.5 播放器状态修复

本阶段重点修复了“切歌后底部歌词/评论打开上一首内容”的问题。

当前约定：

- 播放器维护 `currentTrackId`。
- 底部歌词/评论入口优先使用当前播放 track。
- 当前主内容为歌词/评论时，切歌会自动刷新对应资源。
- `loadMainResource()` 使用请求取消和版本号保护，避免旧请求晚返回覆盖新内容。
- 从完整页面响应中只抽取真正的 `#main-content`，避免页面壳嵌套。

### 2.6 回归测试

新增基础回归测试文件：

```text
tests/test_views.py
```

当前共有 18 个测试，覆盖：

- 核心页面 200 状态。
- 播放量接口。
- 评论 no-cache 响应头。
- 歌词指定 track 解析。
- 上传格式提示和 `.ncm` 排除。

最近一次验证全部通过。

## 3. 当前可演示流程

1. 启动服务：

```powershell
.\.venv\Scripts\python.exe manage.py runserver 127.0.0.1:8000
```

2. 打开首页：

```text
http://127.0.0.1:8000/
```

3. 可演示能力：

- 浏览推荐音乐和最新音乐。
- 从左侧或右侧播放列表选择歌曲。
- 使用底部播放器播放、暂停、拖动进度条、上一首、下一首。
- 打开歌词页并查看当前歌曲歌词。
- 打开评论页并查看当前歌曲评论。
- 上传新音乐并附带歌词。
- 查看播放量递增。

## 4. 已知限制

- 目前还没有真实用户登录系统，用户入口仍偏展示态。
- 搜索框 UI 已存在，但搜索系统尚未产品化。
- 管理后台还没有针对音乐、评论、歌词、用户做完整配置。
- 播放队列仍主要依赖 DOM 顺序，不是真实队列模型。
- 新 GUI 响应式规范尚未完整落地，尤其是手机竖屏全屏播放视图和手机横屏模式。
- 浏览器级前端回归测试还没有自动化，复杂交互仍需要人工验收。

## 5. 下一阶段规划

优先顺序：

1. 落地新 GUI 响应式规范的桌面播放栏和断点统一。
2. 建立真实用户登录系统。
3. 完成搜索系统。
4. 建立真实播放队列和播放模式。
5. 完善评论回复与点赞。
6. 完善 Django 管理后台。
7. 补浏览器级前端回归测试。
