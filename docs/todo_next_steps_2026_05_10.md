## P2 歌词体验

### 9. 新 GUI 响应式设计规范落地

目标：按新的可交互 HTML/CSS 原型统一桌面、手机竖屏、手机横屏三种形态。

关键规则：

- 桌面端 `> 960px`：三列布局，左侧栏 + 主内容 + 右侧上下文。
- 播放栏左右列必须完全对称，使用相同的 `minmax(160px, 1fr)`，中间列固定/弹性居中，禁止右列使用 `auto` 破坏对称。
- JS 的右侧栏显示阈值要和 CSS 断点对齐，重点统一 `960px / 820px / 480px`。
- `<= 820px`：右侧上下文收起为图标/面板，播放 controls 靠右或进入紧凑模式。
- `<= 480px` 竖屏：全屏正在播放视图 + 底部 tab 栏。
- 手机横屏且 `max-height: 500px`：播放栏压缩为单行 `<= 56px`，左侧封面信息，右侧播放列表。
- 进度条视觉可保持细线，但触摸响应区要至少 `24px` 高。
- 主要播放按钮触摸目标 `>= 44px`，次要按钮 `>= 40px`。

验收：

- 桌面端播放/暂停组件在主视觉区域稳定居中，右侧工具区不被挤压。
- `showContext` 的 JS 阈值和 CSS 断点不会互相打架。
- 竖屏手机进入可用的全屏播放视图。
- 横屏手机播放栏单行展示，不挤占主内容高度。
- 进度条拖动区域在触摸设备上可用。
