# Echo 极简音频分享社区 GUI 设计规格说明书

更新时间：2026-05-11

> 本文是 GUI 方向说明。当前代码已经实现桌面三栏应用壳、底部播放器、歌词/评论局部切换等基础能力；新的桌面/竖屏/横屏三形态规范已进入 TODO，后续改造以本文“2026-05-11 新响应式规范”为准。

## 0. 项目定位与核心哲学

- **项目名称**：Echo (Echoes) —— 极简音频分享与评论平台。
- **设计目标**：构建一个类似 Spotify 的“类 SPA (单页应用)”体验。
- **核心特性**：**三栏布局、播放不中断、局部刷新、双色适配、模块化中部。**
- **技术栈**：Django + SQLite3 + Tailwind CSS + HTMX + 原生 JavaScript。

------

## 1. 总体布局规范 (Layout Structure)

### 1.1 PC 端三栏式架构

采用经典的三栏对称布局，确保视觉平衡：

- **左侧栏 (Left Sidebar)**：**280px - 320px**。个人音乐资产管理，支持折叠。
- **中部栏 (Main Content)**：**flex-1 (最大化)**。动态业务交互区。
- **右侧栏 (Context Panel)**：**280px - 320px**。当前音频补充信息，支持折叠/关闭。
- **底部栏 (Playback Bar)**：**100% 宽度，固定高度**。全局静态播放控制器。

### 1.2 对称性与适应性

- **对称原则**：左侧与右侧宽度必须严格一致，确保中部内容在视觉上绝对居中。
- **自适应逻辑**：
  - 左/右栏折叠后，中部栏通过 `transition-all` 平滑扩展填满剩余空间。
  - 底部播放器始终处于视图最底层，不参与任何 HTMX 局部替换。

## 2. 视觉与色彩系统 (Visual & Color System)

### 2.1 品牌色策略

为了拉开视觉档次，我们采用双色调逻辑：

- **品牌识别色 (Brand Color)**：`靛蓝色 (#6366F1)` —— 用于 Logo、登录按钮、主操作、高亮状态。
- **播放状态色 (Playback Color)**：`绿色 (#22C55E)` —— 专门用于播放进度条、音量条、正在播放图标。

### 2.2 深浅色模式适配 (Dark/Light Mode)

| **元素**      | **深色模式 (Dark)** | **浅色模式 (Light)** | **Tailwind 类名示例**                  |
| ------------- | ------------------- | -------------------- | -------------------------------------- |
| **主背景**    | `#121212`           | `#F8F9FA`            | `bg-zinc-950 dark:bg-black`            |
| **卡片/容器** | `#181818`           | `#FFFFFF`            | `bg-white dark:bg-zinc-900`            |
| **主文字**    | `#FFFFFF`           | `#1A1A1A`            | `text-zinc-900 dark:text-white`        |
| **次要文字**  | `#A3A3A3`           | `#6C757D`            | `text-zinc-500 dark:text-zinc-400`     |
| **分割线**    | `#282828`           | `#E9ECEF`            | `border-zinc-200 dark:border-zinc-800` |

## 3. 五大核心组件规范

### 3.1 顶部栏 (Top Bar) —— 导航与身份

- **左侧**：Echo Logo (声波图标 + 字样) + 圆形主页按钮。
- **中间**：胶囊形搜索框。支持 `hx-trigger="keyup changed delay:500ms"` 实时反馈搜索结果至中部栏。
- **右侧**：身份状态切换。
  - *未登录*：展示“注册”与“登录(靛蓝色)”按钮。
  - *已登录*：展示带红点的消息图标、用户圆形头像（点击弹出个人中心下拉菜单）。

### 3.2 左侧栏 (Sidebar) —— 个人资产与历史

- **Library Header**：库图标 + “你的音乐库”文字 + “+”号(快速上传入口)。
- **Filters**：提供“已上传”、“已点赞”两个胶囊形筛选按钮。
- **History List**：按时间倒序排列的最近播放记录。
  - 每条目含：48x48px 圆角封面、标题(粗体)、上传者(灰字)。
- **折叠态**：仅保留图标/封面。悬停时显示 Tooltip 标题。

### 3.3 中部栏 (Main Content) —— 模块化业务区

**必须固定容器 ID：`<main id="main-content">`**。

- **模块化设计**：中部内容由独立组件拼装，不可写死：
  - `Hero Block`：问候语、Banner、大操作按钮。
  - `Grid Block`：方块网格展示歌单/歌手。
  - `List Block`：展示搜索结果或瀑布流音频。
- **详情页模式**：点击音频后进入沉浸模式。
  - **动态背景**：提取封面色生成高斯模糊渐变背景。
  - **同步歌词流**：当前句高亮放大，随播放进度自动垂直滚动。

### 3.4 右侧栏 (Context Panel) —— 沉浸补充

- **视觉媒体组**：1:1 或 4:5 的照片墙/幻灯片，展示 MV 剧照或艺人写真。
- **艺人概览**：圆角卡片，含艺人头像、名称及简短 Bio。
- **上下文信息**：职员表、发布日期、下一首预告。
- **折叠机制**：顶部提供“X”关闭按钮，点击后中部内容区自动向右补齐。

### 3.5 底部播放栏 (Playback Bar) —— 全局底座

- **持久性**：位于 `base.html`，绝不刷新。
- **左侧**：当前曲目微型封面、标题、艺人、心形点赞按钮。
- **中间**：播放控制组（随机、上一首、**播放/暂停(视觉中心)**、下一首、循环）+ 绿色细线进度条。
- **右侧**：音量滑动条、播放队列开关、右侧栏呼出开关。

## 4. 交互技术规范 (Interaction & HTMX)

### 4.1 HTMX 局部刷新逻辑

- **原则**：所有主要页面跳转（首页、搜索、详情、上传、主页）必须定向到 `#main-content`。
- **示例**：`<a hx-get="/track/1/" hx-target="#main-content" hx-push-url="true">`。

### 4.2 播放器通信机制

- **解耦设计**：中部内容区的点击播放按钮，不直接操作音频标签，而是发送 JS 事件。
- **流程**：
  1. 中部按钮触发：`dispatchCustomEvent('echo:play', {src: '...', id: '...'})`。
  2. 底部播放器监听事件并更新 `audio.src`，同时利用 HTMX 请求后端更新 `play_count`。

### 4.3 状态记忆 (localStorage)

系统需本地存储以下状态，确保刷新不丢失：

- `echo_theme`: 'dark' | 'light'
- `echo_sidebar_collapsed`: boolean
- `echo_context_panel_open`: boolean
- `echo_player_volume`: float (0.0 - 1.0)

## 5. 开发建议顺序 (Roadmap)

1. **Phase 1 (骨架)**：在 `base.html` 中用 Tailwind Grid/Flex 完成五大部分的静态布局，实现深浅色切换。
2. **Phase 2 (局部刷新)**：引入 HTMX，完成从首页点击切入详情页且不刷新侧边栏/底部。
3. **Phase 3 (播放器)**：编写全局 `window.EchoPlayer` 对象，实现点击歌曲、底部播放器更新、音乐不断的效果。
4. **Phase 4 (业务模块)**：填充中部栏的模块化组件（网格、列表、歌词同步）。
5. **Phase 5 (细节打磨)**：加入高斯模糊动态背景、折叠动画、音量记忆。

## 6. 2026-05-11 新响应式规范

当前目标：同时兼顾桌面端、手机竖屏、手机横屏三种形态，并解决播放栏居中、触摸目标、横屏高度不足等问题。

### 6.1 断点规范

| 断点 | 布局变化 | 播放栏 | 触摸目标 |
| ---- | ---- | ---- | ---- |
| `> 960px` | 三列：左侧栏 + 主内容 + 右侧上下文 | 三段式，中间居中，左右对称 `1fr` | `32px` 最小 |
| `<= 820px` | 右侧上下文收起为图标/面板 | 隐藏音量区，controls 靠右 | `36px` 最小 |
| `<= 640px` | 左侧栏收起，仅显示图标 | 两列：封面信息 + 控制按钮 | `40px` 最小 |
| `<= 480px` 竖屏 | 全屏播放视图 + 底部 tab 栏 | 封面居中展开，进度条全宽 | `44px` 最小 |
| 手机横屏 | 左：封面+信息，右：播放列表 | 底栏压缩高度 `<= 56px`，合并为一行 | `40px` 最小 |

### 6.2 播放栏居中原则

- 左右列必须完全对称，例如：

```css
grid-template-columns: minmax(160px, 1fr) minmax(280px, 560px) minmax(160px, 1fr);
```

- 禁止右列使用 `auto`，否则右侧工具区会破坏中间控制组的视觉居中。
- JS 中右侧上下文显示阈值必须和 CSS 断点对齐，避免 CSS 与 JS 各自进入不同布局状态。

### 6.3 横屏手机规则

横屏高度通常只有 `360px - 414px`，不适合继续使用顶部栏 + 主内容 + 双行播放栏的常规结构。

建议使用：

```css
@media (orientation: landscape) and (max-height: 500px) {
  /* 单独横屏布局 */
}
```

横屏目标：

- 播放栏压缩为单行，不超过 `56px`。
- 左侧展示封面和歌曲信息。
- 右侧展示播放列表。
- 避免出现需要双手难以触达的操作。

### 6.4 触摸友好规则

- 进度条视觉可以保持 `3px` 左右，但点击响应区域至少 `24px` 高。
- 主播放按钮不小于 `44px`。
- 上一首、下一首、循环、随机等次级按钮不小于 `40px`。
- 横屏模式下禁止出现需要双手复杂操作的隐藏手势。

### 6.5 当前代码状态

- 桌面应用壳和底部播放器已经可用。
- 播放栏已完成主要队列交互：右侧播放列表由 `playQueue` 驱动，歌曲行/卡片右键菜单可加入队列和下一首播放。
- 播放栏左右对称列、手机竖屏全屏播放视图、手机横屏模式、统一断点常量仍是后续任务。
